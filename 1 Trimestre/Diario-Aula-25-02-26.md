## Atividade 01 — Ciclo de Vida do Desenvolvimento de Software

### Exemplos de ciclos de desenvolvimento de software

- **Modelo em cascata**:  
  Requisitos → Arquitetura/Design → Implementação → Verificação → Manutenção.  
  Modelo linear, bom para pequenos projetos que não possuem grande escalabilidade. Há contato com o cliente, mas não envolvimento ao longo do processo.

  - **Modelo evolutivo**:  
  Modelo em que os requisitos e o código vão evoluindo a cada etapa, criando uma espécie de espiral. O documento de requisitos só fica pronto no final do projeto, fazendo com que, em cada estágio, o documento de requisitos possa sofrer mudanças, mantendo o software sempre em evolução até que o usuário não sugira mais requisitos.

- **Modelo incremental**:  
  Parecido com o modelo em cascata, porém possui várias etapas. Após implementar, verificar e fazer a manutenção, retornamos para os requisitos e design novamente.  
  A ideia desse modelo é quebrar o software em várias tarefas e incrementar uma de cada vez, sendo assim mais próximo dos princípios da metodologia ágil. Se difere do espiral pela entrega a cada etapa.

## Manifesto Ágil

A metodologia ágil é uma maneira de criar um sistema envolvendo o usuário e, principalmente, realizando entregas contínuas.  
A cada entrega, o cliente tem a oportunidade de dar feedback sobre o que foi feito.
Ideologias:

- Pessoas.
- Software executando.
- Entender o cliente.
- Aceitar e entender as mudanças.

Esse modelo permite maior flexibilidade e adaptações de acordo com as necessidades do usuário.  
A principal ideia dessa metodologia é manter o software alinhado com as ideias do cliente desde o planejamento até a finalização.
